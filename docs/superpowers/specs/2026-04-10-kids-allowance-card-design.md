# 어린이 용돈카드 서비스 — 설계 문서

**작성일**: 2026-04-10  
**상태**: 브레인스토밍 완료 (추가 구체화 반영)

---

## 1. 서비스 개요

어린이(8~13세)가 실물 선불카드와 모바일 앱을 통해 용돈을 직접 관리하며 경제관념을 기르는 서비스.
부모가 충전 및 한도를 설정하고, 아이는 카드로 결제하면서 지출을 스스로 파악한다.

### 핵심 가치
- **자율성**: 아이가 스스로 잔액을 확인하고 소비를 결정
- **투명성**: 부모와 아이 모두 지출 이력을 실시간으로 공유
- **안전성**: 부모가 한도·업종을 제어하여 과소비 방지

---

## 2. 사용자 및 구조

### 대상 사용자
- **아이**: 초등 중고학년 (8~13세)
- **부모**: 아이의 용돈을 관리하는 보호자

### 앱 구조
- **단일 앱, 멀티 프로필 전환 방식**
- 부모 계정 1개에 자녀 프로필 N개 연결 (형제자매 대응)
- 로그인 후 프로필 선택으로 부모↔아이 화면 전환

### 카드
- **실물 선불카드** 발급 (카드사 파트너사 연동)
- 앱과 1:1 연동, 충전 잔액 범위 내에서만 사용 가능

---

## 3. 핵심 기능

### 3-1. 부모 화면

| 기능 | 설명 |
|------|------|
| 자녀 현황 대시보드 | 자녀 목록, 각 아이의 잔액·이번 주 사용액 한눈에 확인 |
| 용돈 충전 | 정기 자동 충전 (매주/매월) + 비정기 즉시 충전 |
| 한도 설정 | 1회 / 일별 / 주별 최대 사용 금액 설정 |
| 카테고리 제한 | 특정 업종(게임, 주류 등) 결제 자동 차단 |
| 지출 이력 모니터링 | 자녀의 모든 결제 이력 조회 |
| 알림 설정 | 한도 80% 도달, 차단 업종 시도, 결제 발생 시 푸시 알림 |
| 지출 리포트 | 주간/월간 카테고리별 지출 요약 수신 |

### 3-2. 아이 화면

| 기능 | 설명 |
|------|------|
| 잔액 메인 표시 | 현재 잔액을 크고 직관적으로 표시 |
| 한도 시각화 | 이번 주 사용 금액 / 남은 한도를 프로그레스 바로 표시 |
| 지출 이력 타임라인 | 가맹점명, 금액, 시간 순으로 카드 형태 표시 |
| 카테고리 자동 분류 | 식비, 문구, 편의점, 교통 등 자동 태깅 |
| 메모 추가 | 아이가 지출 항목에 직접 메모 입력 가능 |
| 바텀 탭 | 잔액 / 이력 / 설정 (3개 이하) |

### 3-3. 카드 연동

- 결제 시 실시간 앱 알림 + 이력 자동 기록
- 분실 시 앱에서 즉시 카드 잠금/해제
- 잔액 부족 시 결제 거절 (선불 특성상 마이너스 불가)

---

## 4. UX 원칙

### 아이 UX
- 텍스트보다 숫자와 아이콘 중심
- 잔액이 줄어들 때 색상 변화로 시각적 피드백 (초록 → 노랑 → 빨강)
- 복잡한 메뉴 없이 바텀 탭 3개 이하
- 결제 이력은 타임라인 형태로 직관적 표현

### 부모 UX
- 한 화면에서 자녀 전체 현황 파악
- 충전은 3탭 이내로 완료 가능
- 이상 지출 알림으로 능동적 모니터링 가능

---

## 5. 수익 모델

- **앱 자체는 무료** 제공
- **카드 발급비** 1회 청구 (금액 추후 확정, 예상 5,000~10,000원)
- 카드 재발급(분실 등) 시 추가 발급비 청구

---

## 6. 서비스 네이밍 방향

- **신뢰감 중심 네이밍**: 부모가 믿고 맡길 수 있다는 인상
- 예시 방향: 패밀리페이, 키즈월렛, 용돈안심 등 (최종 확정 전 브랜딩 팀 협의 필요)

---

## 7. 가입 및 온보딩

### 인증 방식
- 부모 **공인인증서 / 금융인증서** 기반 가입 (미성년자 금융서비스 법적 요건 충족)
- 아이는 별도 인증 없이 부모 계정의 서브 프로필로 생성

### 온보딩 흐름
1. 부모 본인인증 (금융인증서)
2. 자녀 프로필 생성 (이름, 나이)
3. 카드 발급 신청 (배송지 입력)
4. 첫 충전 및 한도 설정
5. 카드 수령 후 앱에서 카드 등록 → 사용 시작

---

## 8. 충전 수단

| 수단 | 설명 |
|------|------|
| 간편결제 연동 | 카카오페이, 토스, 네이버페이 등 연동 |
| 자동이체 | 부모 은행 계좌에서 정기일에 자동 차감 |

---

## 9. 알림 정책

| 상황 | 수신자 | 방식 |
|------|--------|------|
| 카드 결제 발생 | 부모 + 아이 | 푸시 알림 실시간 |
| 차단 업종 결제 시도 | 부모 | 푸시 알림 즉시 |

---

## 10. 카드 분실 대응

1. 아이 또는 부모가 앱에서 **즉시 카드 잠금**
2. 앱 내에서 **재발급 신청** (별도 CS 연락 불필요)
3. 재발급 카드 배송 완료 후 앱에서 새 카드 등록

---

## 11. 기술 고려사항

| 항목 | 내용 |
|------|------|
| 카드 연동 | 선불카드 파트너사 카드사 API 연동 필요 |
| 실시간 알림 | 결제 발생 시 푸시 알림 + 앱 내 피드 동시 제공 |
| 본인인증 | 부모 공인인증서/금융인증서 연동 |
| 간편결제 | 카카오페이·토스·네이버페이 API 연동 |
| 자동이체 | 은행 오픈뱅킹 API 연동 |
| 개인정보 | 아이 데이터 최소 수집 원칙 적용 |
| 보안 | 앱 잠금 (PIN/생체인증), 카드 원격 잠금/해제 |
